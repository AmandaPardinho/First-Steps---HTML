Main tags and how to use them

Home | About Me | Contact

body: place where all content is inserted.

main: the main content of a page; it is unique and exclusive to that page.

main: the main content of a page; it is unique and exclusive to that page.

br: line break.

header: works both in the body and in the sections.

footer:
-place to put HTML entities;
-works both in the body and in the sections.

section:
-any part of a document
-each section has its own content;
-these contents can be separated as needed.

article:
-kind of section that makes sense on its own;
-is self-contained.

nav:
-navigation menu;
-can create an internal navigation menu (within the site itself) or an external one (to another site).

aside:
-shows that a certain content is not substantial for what is delivered on that page;
-is complementary information;
-can be used to place advertisements on a website.

div:
-non-semantic tag;
-should be used as little as possible in relation to the other tags.
-should be used when you want to make a subdivision in the semantic content, but there is no specific tag for this;
-is a content block;
-it occupies the space of this block from beginning to end (from one edge to the other);
-divides the text into blocks;
-all semantic content is a div;
-are div that have been given a name and a specific function.

span:separates a piece of text when you want to keep it in the same block.

headline:
-describe what the section does;
-can be used in any area of the site, being particularly important in articles and sections;
-goes from h1 to h6.

paragraph: assists in defining the normal text flow.

links:
-are the reference from one page to another page;
-are defined as anchor links;
-anchor: the point to which you are linking and that can be a letter, a word, or an image;
-you use the "href" inside the link tag to reference the path that must be followed;
-ways to type the link path:
-with nothing: related to the current location;
-../: one level higher;
-/: current root;
-protocol + url;
-https://;
-//url;
-target: says the location where you want to open the link;
-optional component;
-good practice: do not "break" the back button, i.e. do not open the page in a new tab (Did I do that? Of course not.>] );
-_self: opens in the same tab;
-_blank: opens in another tab;
-_parent: used when you are in an iframe inside another page (popup?).

Note 1: HTML serves to structure the site (it is the skeleton of the site). The appearance of the site is implemented by CSS.

Note 2: When sections are not well defined, a space is opened up for the browser to interpret them as it wishes. This can lead to problems in structuring the site.

Note 3: The syntactically correct way to use break is "br /" (it is a self-closing tag).

Note 4: Today, visual changes are no longer made by HTML code. All visual changes are made in CSS.
